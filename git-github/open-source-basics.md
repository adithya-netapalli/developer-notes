# Open Source Contribution

## What is Open Source?

Open Source projects are publicly available projects whose source code can be viewed and contributed to by anyone.

Examples:
- React
- VS Code
- Node.js

## How Contributions Work

1. Find an issue.
2. Make the required changes.
3. Create a Pull Request (PR).
4. Maintainers review the changes.
5. The PR is approved and merged.

## Fork a Repository

A fork is a copy of someone else's repository in your GitHub account.

It allows you to work on the project without modifying the original repository.

## Clone a Repository

```bash
git clone repository-url
```

Downloads the repository to your local system.

### Example

```bash
git clone https://github.com/user/project.git
```

## Make Changes

After cloning:

- Open the project.
- Make the required changes.
- Save the files.

## Commit Changes

```bash
git add .
git commit -m "fix issue"
```

Commits the changes locally.

## Push Changes

```bash
git push
```

Pushes changes to your forked repository.

## Create a Pull Request

1. Open your forked repository.
2. Click **Compare & Pull Request**.
3. Add details about the changes.
4. Submit the Pull Request.

## Keep Fork Updated

```bash
git remote add upstream repository-url
```

Adds the original repository as an upstream remote.

## Good First Issues

Many projects provide:

```txt
Good First Issue
```

These issues are beginner-friendly and suitable for first-time contributors.

## Summary

- Open Source projects are publicly available.
- Fork → Clone → Change → Commit → Push → Pull Request.
- Maintainers review and merge contributions.
- Good First Issues are a good starting point for beginners.
