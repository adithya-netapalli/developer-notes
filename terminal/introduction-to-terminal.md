# Introduction to Terminal and Shell

## GUI vs CLI

We can interact with a computer in mainly 2 ways:

### 1. GUI (Graphical User Interface)
- Uses buttons, windows, icons, and mouse clicks.
- Example:
  - File Explorer
  - Chrome Browser

### 2. CLI (Command Line Interface)
- Uses text commands to interact with the computer.
- Commands are typed inside the terminal.

---

## What is a Terminal?

A terminal is a place where we type commands to communicate with the computer.

Instead of clicking buttons, we can directly tell the computer what to do using commands.

### Definition Flow

```text
A user types a command in the terminal.
The shell reads and understands the command.
The operating system executes the command and gives the output.
```

---

## What is a Shell?

A shell is a program that reads the commands typed in the terminal and passes them to the operating system.

It acts like a bridge between:
- User
- and Operating System

---

## Why Developers Use Terminal

- Faster than GUI for many tasks
- Useful for automation
- Important for Git, Node.js, Docker, AWS, etc.
- Helps manage files and projects efficiently
