# Semantic Tags

## What is Semantic?

Semantic means giving meaning to content.

Semantic tags describe the purpose of the content they contain.

They help developers, search engines, and screen readers understand the structure of a webpage.

## Semantic Block Elements

Semantic block elements work like `<div>` but provide meaning to the content.

### Common Semantic Block Elements

- `<header>` - Defines the header section of a webpage.
- `<nav>` - Contains navigation links.
- `<main>` - Contains the main content of the webpage.
- `<section>` - Represents a specific section of content.
- `<footer>` - Defines the footer section of a webpage.

### Example

```html
<header>
  <h1>My Website</h1>
</header>

<nav>
  <a href="#">Home</a>
  <a href="#">About</a>
</nav>

<main>
  <section>
    <h2>About Us</h2>
    <p>Content goes here...</p>
  </section>
</main>

<footer>
  Copyright 2026
</footer>
```

## Semantic Inline Elements

Semantic inline elements add meaning to text.

### Common Semantic Inline Elements

- `<em>` - Emphasizes text.
- `<strong>` - Defines important text.

### Example

```html
<p>
  Learn <em>HTML</em> and <strong>CSS</strong>.
</p>
```

## Why Use Semantic Tags?

- Improves code readability.
- Helps search engines understand content.
- Improves accessibility for screen readers.
- Makes website structure easier to understand.
- Considered a best practice in HTML.

## Summary

- Semantic means giving meaning to content.
- Semantic tags describe the purpose of content.
- Semantic block elements provide structure to webpages.
- Semantic inline elements add meaning to text.
- Semantic HTML improves readability, accessibility, and SEO.
